GitHub & SSH Protocol

Using the SSH protocol, you can connect and authenticate to remote servers and services. With SSH keys, you can connect to GitHub without supplying your username or password at each visit.
more docs, click here.

> After Creating a new account in github.com

> you can access to Github SSH Protocol services

// checking for existing SSH keys
$ ls -al ~/.ssh

/*/ Generating a new SSH key /*/
//Paste the text below, substituting in your GitHub email address.
$ ssh-keygen -t rsa -b 4096 -C "your_email@example.com"

#This creates a new ssh key, using the provided email as a label.

#When you're prompted to [ Enter a file in which to save the key ], press Enter. This accepts the default file location.

#At the prompt, type a secure passphrase. For more information, see [ Working with SSH key passphrases ].
/*/	end section /*/


/*/ Adding your SSH key to the ssh-agent /*/
//start the ssh-agent in the background.
$ eval "$(ssh-agent -s)"

//add your SSH private key to the ssh-agent and store your passphrase in the keychain
$ ssh-add -K ~/.ssh/id_rsa
#then add the SSH key to your GitHub account.
/*/	end section /*/


/*/ Adding a new SSH key to your GitHub account /*/
//copy the SSH key to your clipboard.
$ pbcopy < ~/.ssh/id_rsa.pub
#copies the contents of the id_rsa.pub file to your clipboard
/*/	end section /*/

/*/ Testing your SSH connection /*/
//enter the following:
$ ssh -T git@github.com
#Verify that the fingerprint in the message you see matches one of the messages in step 2, then type
#Verify that the resulting message contains your username. If you receive a ( permission denied ) message, see [ Error: Permission denied (publickey) ].
/*/	end section /*/


/*/ Deleting SSH keys terminal /*/
$ rm ~/.ssh/{filename} // exemple: rm ~/.ssh/id_rsa.pub
# there is three files created by default {id_rsa, id_rsa.pub, known_hosts}
